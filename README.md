# 1 献给想要有自己网站的人

我一直很想要一个我自己的网站，这个想法是在我初一的时候就有的。

但是我的第一个网站是在高中时才有的，这个网站使用的是纯粹的 HTML，CSS，Javascript 来写的。

尽管我在这个网站上面写了很多东西，C# 笔记，Blender 笔记，MC 数据包教程等等，但是实际上大量的时间都花费在了 HTML 上。

那个时候，我还不知道如何把 Markdown 转化为 HTML，所以一度使用 HTML 来写笔记。随后，理所当然的，这个网站在运行了一段时间之后，关闭了。

现在我上了大学，决定重新捡起来这个网站。所以有了这一篇文章。谨以此文，献给每一个想要有自己的小天地的孩子。

## 1.1 项目的优势以及劣势

网站使用了 `Jekyll+Liquid` 的方式来渲染，整个网站都是使用 `Markdown` 来渲染的。

使用这个框架的目的使简化网站的搭建，使用 `Markdown` 可以简化在制作网站过程中的成本。

但是随着笔者笔记的上传以及日志的完善，发现并不是非常方便，尽管相较于 `HTML` 而言，`Markdown` 降低了一定的成本，但是我们也需要承认，`Jekyll` 支持 `Latex`，`mermaid` 等等都不是非常方便，需要我们在一定程度上去学习这些东西。

此外，对于部分的需求，例如折叠文本。

对于 `HTML` 而言，仅仅是两个标签的问题而已。

```html
<details>
    <summary>点击展开</summary>
    折叠文本
</details>
```

但是如果在 `Markdown` 中使用这个标签，会导致后续内容以及内部内容的渲染失误，`kramdown` 引擎 (`jekyll` 的默认 `Makrdown` 渲染引擎) 会默认 `HTML` 内部的文本是 `HTML`。这意味着我们无法在 `HTML` 中使用 `Markdown` 语法。这与我们的目的截然相反。

为了解决这个问题，我尝试了两个方式。

**方式一**

我使用了 `_plugins` 文件夹，并写了一 `ruby` 的代码，声明了一个新的 `Liquid` 标签。但是 `GitHub Page` 不允许我们使用第三方的标签，这意味着我们无法在这个网站中使用这个技术。(普通的网站是可以使用的)

**方式二**

我随后使用了 `_includes` 文件夹，添加了一个引入文件。然后使用了 `{{ capture }}` 捕捉内容，放入文件中。并制作了 `HTML` 和 `CSS` 文件，才解决了这个问题。

换句话说，这个技术仅仅是**降低了使用 `HTML & CSS` 的频率，而不是不使用这些技术**。当然了，如果你对这个网站的要求较少，那么自然不必多说，你可以使用第三方的模板，这样就完全没有这个问题了。

不过我比较喜欢自定义这个网站，所以我决定自己写以上的一切。这个网站，也同样献给和我一样想法的人。

## 1.2 前置要求

1. 会使用 `markdown` 标记语言。
   1. 这个是**基础之中的基础**了，其实严格来说，并不是搭建网站的需求，而是写文章的需求，进一步的，你可能也需要会 `Mermaid` 和 `LaTex` 等等。
2. 有一定的 `HTML & CSS` 功底
   1. 这个项目对于如上两个技术的要求很低，但是并不是没有要求。毕竟我们想要写出一个好看的，符合我们自己心意的网站。
   2. 那么是不是我们需要熟练的使用如上两个技术来写某个东西呢? 并不是，仅仅需要我们可以看得懂相关的代码即可。换句话说有**阅读能力**即可。
3. 电脑
   1. 这是你搭建网站的前提，使用手机自然不用多想了。
4. 一定的电脑基础知识
   1. 本项目需要安装一些配置库，这些东西并不是我会教你的，所以你需要有一定的电脑的基础知识，比如如何打开终端等等。
   2. GitHub 的基础知识，你需要会一点 GitHub 的操作。
5. **耐心**以及**热情**
   1. 这两个东西是重中之重，所以作为压轴出场，如果有以上两者，成功率会大大提高。
   2. 去试错吧，去大量的尝试，然后反复的修改，最后一定会得到一个满意的网站。

## 1.3 你会学到什么

1. 使用 GitHub 搭建一个网站，不使用 Jekyll 和 Liquid 这个技术。
2. Jekyll 和 Liquid 使用。
3. Ruby，HTML，CSS 的基础使用
4. 使用一门语言实现一定程度上的自动化 (笔者使用的是 `cpp`)
